# 1. LogicPilot

## Think Before You Argue

---

### 한 줄 설명

> LogicPilot은 표현하기 전에 생각을 만들고 검증하는 Inquiry · Reasoning · Critical Thinking Foundation Engine이다.

---

### 왜 필요한가

많은 사람이 생각이 충분히 형성되기 전에 말하고, 쓰고, 주장한다. 표현의 문제가 아니라 사고의 문제다. LogicPilot은 이 단계를 해결한다.

표현하기 전에:
- 좋은 질문을 만들었는가?
- 사실과 의견을 구분했는가?
- 핵심 쟁점을 발견했는가?
- 판단의 근거를 구성했는가?

---

### 핵심 기능

#### Inquiry — 질문 능력
- 좋은 질문 생성
- 문제 정의
- 숨겨진 전제 탐색
- 다양한 관점 확장

#### Reasoning — 생각 형성
- 정보 연결
- 원인 분석
- 가설 생성
- 자신의 관점 형성

#### Case Draft — Critical Thinking Foundation
| 단계 | 내용 |
|---|---|
| 상황 분석 | 문제의 맥락과 배경 이해 |
| 사실/의견 구분 | 객관적 근거와 주관적 판단 분리 |
| 핵심 쟁점 발견 | 토론의 실제 핵심이 무엇인지 파악 |
| 이해관계 분석 | 누가 영향을 받는가 |
| 선택지 비교 | 가능한 입장과 논거 검토 |
| 판단 근거 구성 | 내 주장을 뒷받침할 근거 정리 |

---

### 목표 역량

- Analytical Critical Thinking
- 독립적 사고 형성 능력
- 표현 이전의 사고 구조화 능력

---

### ICD 위치

```
[Inquiry] → [Reasoning] → [C 초입] → Deliberation → Reflection
     ↑ LogicPilot의 영역 ↑
```

---

### 설계 원칙 적용

- **Person-centred**: AI가 답을 주지 않는다. 사용자가 스스로 생각을 구성한다
- **Offline-oriented**: 화면이 아닌 실제 사고 과정을 훈련한다
- **Face-to-face 준비**: LogicPilot에서 만든 생각은 CODA에서 사람과 검증된다

# 2. Motion Game

## Structure Before You Debate

---

### 한 줄 설명

> Motion Game은 생각을 논증 가능한 구조로 변환하는 Argumentative Critical Thinking Training Engine이다.

---

### 왜 필요한가

생각이 있어도 논증 구조가 없으면 설득하지 못한다. Motion Game은 LogicPilot에서 형성된 생각을 실제 주장 구조로 훈련하는 환경이다.

논증에는 규칙이 있다:
- 명제(Motion)가 무엇인가?
- 핵심 개념을 어떻게 정의하는가?
- 어디까지 이야기할 것인가?
- 반론을 어떻게 예측하는가?
- 내 주장의 입증 책임은 무엇인가?

---

### 핵심 경험 요소

| 요소 | 내용 |
|---|---|
| **Motion** | 토론 명제 이해 — 이 토론이 무엇에 관한 것인가 |
| **Definition** | 핵심 개념 정의 — 용어를 어떻게 규정하는가 |
| **Scope** | 토론 범위 설정 — 어디까지 다룰 것인가 |
| **Argument** | 주장 구성 — 내 입장과 근거를 구조화한다 |
| **Counterargument** | 반론 예측 — 상대방이 어떻게 공격할 것인가 |
| **Burden of Proof** | 입증 책임 검토 — 내가 무엇을 증명해야 하는가 |

---

### 목표 역량

- Argumentative Critical Thinking
- 생각을 논리적 주장 구조로 만드는 능력
- 반론을 예측하고 대응하는 능력

---

### ICD 위치

```
Inquiry → Reasoning → [Critical Thinking] → Deliberation → Reflection
                            ↑ Motion Game의 영역 ↑
```

---

### 설계 원칙 적용

- **Person-centred**: 논증 구조를 AI가 만들어주지 않는다. 사용자가 직접 구성한다
- **Offline-oriented**: CODA에서의 실제 대면 토론을 위한 준비 단계
- **Face-to-face 준비**: Motion Game에서 훈련한 구조는 CODA의 Human vs Human에서 실전 검증된다

# 3. CODA

## Test Through Human Interaction

---

### 한 줄 설명

> CODA는 생각을 실제 인간 상호작용 속에서 검증하고 개선하는 Human Deliberation Practice Engine이다.

---

### 왜 필요한가

아무리 잘 만든 생각도 사람과의 실제 상호작용 속에서 검증되지 않으면 성장하지 않는다. CODA는 단순한 말하기 훈련이 아니다. LogicPilot과 Motion Game에서 형성·구조화한 생각을 실제 인간 관계 속에서 표현하고, 충돌시키고, 개선하는 과정이다.

---

### 핵심 경험 구조

```
Claim → Evidence → Counterargument → Rebuttal → Deliberation
주장      근거         반론              재반론       숙의
```

---

### 운영 모드

#### Mode 1 — AI vs AI (Observer)
- AI 간 토론을 관찰하며 숙의 과정과 논증 구조를 학습한다
- 처음 참여하는 사용자의 진입 단계

#### Mode 2 — AI vs Human (Practice)
- 개인이 AI를 상대로 자신의 생각을 표현하고 연습한다
- 안전한 환경에서 논증 능력을 단련한다

#### Mode 3 — Human vs Human (Participate)
커뮤니티 숙의(Community Deliberation) 모드.  
두 사람의 토론이 아닌, **주변 모든 사람이 숙의 과정에 참여하는 경험**이다.

구조:
```
Opening → Audience Feedback
        → Argument → Audience Feedback
        → Crossfire → Audience Feedback
```

Audience Reaction System:
| 반응 | 의미 |
|---|---|
| Strong Argument | 설득력 있었다 |
| Needs More Evidence | 근거가 부족하다 |
| Clear Explanation | 설명이 명확했다 |
| Emotional Impact | 감정적으로 공감됐다 |
| Interesting Perspective | 새로운 관점이었다 |

AI Moderator 역할: 토론 관리 · 시간 관리 · 규칙 관리 · 관객 참여 촉진

#### Mode 4 — Guided Coach Team Mode (Collaborate)
AI가 Coach 및 Facilitator 역할을 수행하는 팀 기반 숙의 환경.

프로세스:
```
세션 생성 (Topic · 난이도 · 형식 설정)
    ↓
팀 구성 (링크 참여 → 역할 배정 → 팀 편성)
    ↓
AI 퍼실리테이션 (Round · Time · Participation 관리)
    ↓
발언 지원 (Hint Card · Feedback Card · Objection System)
```

Hint Card: 정답이 아닌 방향을 제시 (Opening: 핵심 주장 제시 / Rebuttal: 상대 논거 약점 찾기)  
Feedback Card: 발언의 논리 연결성·라운드 적합성·설득력·명확성 평가

---

### Moderator Card — Control Layer

CODA 모든 모드의 핵심 설계 도구. 토론 전에 목적과 구조를 정의한다.

| 항목 | 내용 |
|---|---|
| Purpose | 왜 토론하는가 (학습 / 정책 검토 / 의사결정 / 아이디어 탐색) |
| Desired Outcome | 어떤 결과를 원하는가 (최종 결정 / 추천안 / 관점 탐색 / 리스크 분석) |
| Stakeholders | 누가 영향을 받는가 |
| Key Questions | 반드시 다뤄야 할 질문 |
| Constraints | 예산 · 시간 · 정책 · 법률 등 고려 조건 |
| Evaluation Criteria | 효과성 · 실행 가능성 · 공정성 · 지속 가능성 |

Moderator Card는 Manual 생성 또는 AI 즉석 생성(Moderator Wizard) 두 가지 방식으로 만들 수 있다.

---

### Red Team Mode — Critical Thinking Stress Test

상대를 이기는 것이 목적이 아니다. 내 생각의 약점을 찾는 것이 목적이다.

핵심 질문:
- "내 생각이 틀릴 수 있는 이유는 무엇인가?" (지적 겸손)
- "가장 강한 반대 논리는 무엇인가?"
- "빠진 관점은 없는가?"

---

### CODA Architecture

```
Layer 1: Moderator Card (Why)
        ↓
Layer 2: Deliberation Mode (How)
        ↓
Layer 3: Debate Phase (Process)
        ↓
Layer 4: Outcome (Result)
```

---

### 목표 역량

- 의견 표현 능력
- 상대 관점 이해 능력
- 반론 대응 능력
- 설득과 협력 능력
- 집단적 숙의 경험
- 사회적 신호(Social Cues) 해석 능력

---

### ICD 위치

```
Inquiry → Reasoning → Critical Thinking → [Deliberation] → Reflection
                                                ↑ CODA의 영역 ↑
```

---

### 설계 원칙 적용

- **Person-centred**: AI가 결론을 내리지 않는다. 인간이 숙의를 통해 판단한다
- **Offline-oriented**: 사람들이 화면을 보는 것이 아니라 서로를 바라보게 한다
- **Face-to-face**: 표정·감정·분위기 등 사회적 신호를 포함한 실제 대면 상호작용을 중심에 둔다

> CODA는 단순한 Debate App이 아니다.  
> **Structured Deliberation을 설계하고 운영하는 Human Reasoning OS**이다.
